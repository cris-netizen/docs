Esta página es tu diccionario de referencia rápida. Aquí encuentras todos los términos que usamos en la documentación de Puente OS — desde los conceptos de gestión hasta los componentes de construcción de software. Si dudas sobre el significado de alguna palabra mientras lees, vuelve aquí.

| Término | Definición |
|---|---|
| **Workspace** | Contenedor principal de tu organización. Una empresa = un workspace. |
| **Team** | Sub-grupo dentro del workspace, típicamente un departamento o squad. |
| **Member** | Persona con acceso al workspace. Puede pertenecer a uno o varios Teams. |
| **Project** | Iniciativa operativa concreta. Contenedor donde conviven tareas y construcción. |
| **Initiative** | Objetivo estratégico que cruza varios Teams y Projects. |
| **Issue** | Tarea de gestión. Tiene estado, prioridad, asignado y fecha límite. |
| **Sub-issue** | Tarea hija de un Issue. Te sirve para descomponer trabajo en pasos más pequeños. |
| **Cycle** | Período recurrente (1-8 semanas) donde el Team enfoca su trabajo y mide su velocidad. |
| **View** | Lente dinámica para filtrar y ordenar Issues o Builds según los criterios que elijas. |
| **Build** | Artefacto de software construido dentro de un Project. Hay 4 tipos: App, Table, Workflow y Agent. |
| **App** | Interfaz web construida con el App Builder. La capa visual con la que interactúa el usuario final. |
| **Table** | Datos persistentes con schema. Equivalente a una base de datos sin escribir SQL. |
| **Workflow** | Automatización visual con triggers, condiciones, código Python y agentes encadenados. |
| **Agent** | Agente IA conversacional. Puedes embeber el agente como widget en cualquier sitio externo. |
| **Trigger** | Evento que dispara un Workflow (p. ej., cambio en una fila, llegada de un email). |
| **Node** | Cada paso dentro de un Workflow. Puede ser de tipo acción, código, agente o trigger. |
| **Scenario** | Otra forma de llamar a un Workflow. Tienen el mismo significado en la interfaz. |
| **Schema** | Estructura de columnas (nombre + tipo de dato) que define una Table. |
| **Prompt** | Texto que le proporcionas al agente o al App Builder para que genere un resultado. |
| **Modo Inteligente** | Modo de configuración del agente donde la IA genera el prompt por ti automáticamente. |
| **Modo Manual** | Modo de configuración donde tú escribes el prompt directamente. |
| **Modo Inspect** | En el App Builder, modo que te permite editar la app haciendo clic en cualquier elemento. |
| **Preview** | Vista de la app generada en tiempo real, antes de publicarla para tus usuarios. |
| **Theme** | Paleta de colores, tipografías y bordes que aplica de forma global a tus Apps. |
| **Widget** | Componente embebible — típicamente el chatbot de un Agent — que puedes insertar en cualquier sitio externo. |
| **Variable `{{ }}`** | Sintaxis para pasar datos entre Nodes dentro de un Workflow. |
| **Bookmark** | Nodo o configuración guardada que puedes reutilizar en otros Workflows. |
| **Triage** | Bandeja de entrada donde llegan los Issues sin asignar para que los revises y rutees al Team correspondiente. |
